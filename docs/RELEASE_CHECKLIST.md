# Release Checklist

Do not cut a release unless all of the following are true.

## Required

- CI is green.
- Relevant local validation passes.
- Docs match current behavior.
- Examples still work.
- Security posture was not weakened accidentally.

## Release notes should answer

- What changed?
- Who is affected?
- Is anything breaking?
- What should users test after upgrading?

## Before publishing

- Confirm the versioning decision is explicit.
- Call out any contract changes clearly.
- Note any follow-up work that did not make the release.
